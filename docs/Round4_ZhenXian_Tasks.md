### Day 1-2: 端到端测试设计

1. **设计5个完整用户旅程测试**

   **场景1: 告警诊断完整旅程**
   ```
   用户查询INS-001的CPU告警
   → Orchestrator接收
   → DiagnosticAgent诊断
   → AlertCorrelator关联分析
   → 知识库检索SOP
   → 生成诊断报告
   验证: 响应<30秒，报告完整
   ```

   **场景2: 会话恢复与上下文**
   ```
   用户新建会话查询INS-001状态
   用户新建会话查询最近告警
   → 识别为同一用户
   → 复用上下文
   验证: 无需重新认证，历史可访问
   ```

   **场景3: 错误注入与降级**
   ```
   触发: API超时错误注入
   用户查询状态
   → 检测超时
   → 触发重试
   → 降级响应
   验证: 友好错误提示，审计日志记录
   ```

   **场景4: 批量告警关联分析**
   ```
   输入: 10个活跃告警
   → AlertCorrelator分析
   → 构建告警链
   → 识别根因
   → 生成收敛建议
   验证: 根因正确，告警链完整
   ```

   **场景5: 工单创建与审批**
   ```
   用户请求创建重启工单
   → RiskAssessmentAgent评估
   → 确认为L3风险
   → 生成审批请求
   验证: 风险评估正确，审批流触发
   ```

2. **建立性能基线测试用例**
