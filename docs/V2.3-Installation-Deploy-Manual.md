## 二、一键部署（推荐）

### 2.1 macOS/Linux一键部署

```bash
# 下载或克隆项目
git clone https://github.com/chongjie-ran/Javis-DB-Agent.git
cd Javis-DB-Agent

# 执行一键部署脚本
chmod +x install.sh
./install.sh
```

### 2.2 验证安装

```bash
# 检查Python版本
python3 --version  # 应显示 3.10+

# 检查依赖
pip list | grep -E "fastapi|sqlglot|chromadb"

# 启动服务
python3 -m uvicorn src.main:app --host 0.0.0.0 --port 8000

# 访问Dashboard
open http://localhost:8000/dashboard/
```

## 三、手动部署

### 3.1 安装Python环境

**macOS**：
```bash
# 使用Homebrew安装Python
brew install python@3.11

# 创建虚拟环境
python3.11 -m venv .venv
source .venv/bin/activate
```

**Ubuntu/Debian**：
```bash
# 安装Python和依赖
sudo apt update
sudo apt install python3.11 python3.11-venv python3-pip

# 创建虚拟环境
python3.11 -m venv .venv
source .venv/bin/activate
```

### 3.2 安装Ollama（LLM推理）

```bash
# macOS/Linux安装
curl -fsSL https://ollama.com/install.sh | sh

# 下载模型
ollama pull glm4  # 智谱模型

# 验证
ollama list
```

### 3.3 安装PostgreSQL

**macOS**：
```bash
brew install postgresql@15
brew services start postgresql@15
```

**Ubuntu**：
```bash
sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql
```

**创建数据库**：
```bash
# 连接PostgreSQL
psql -U postgres

# 创建数据库
CREATE DATABASE javis_db;

# 创建用户（可选）
CREATE USER javis_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE javis_db TO javis_user;
```

### 3.4 安装项目依赖

```bash
# 克隆项目
git clone https://github.com/chongjie-ran/Javis-DB-Agent.git
cd Javis-DB-Agent

# 激活虚拟环境
source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 或安装所有依赖（包括开发依赖）
pip install -e ".[dev]"
```

### 3.5 配置环境变量

```bash
# 创建.env文件
cat > .env << EOF
# 数据库配置
JAVIS_PG_HOST=localhost
JAVIS_PG_PORT=5432
JAVIS_PG_USER=postgres
JAVIS_PG_PASSWORD=
JAVIS_PG_DATABASE=javis_db

# Ollama配置
OLLAMA_HOST=http://localhost:11434
OLLAMA_MODEL=glm4

# API模式（mock/real）
JAVIS_API_MODE=mock
EOF
```

## 四、Docker部署（可选）

### 4.1 构建镜像

```bash
# 构建镜像
docker build -t javis-db-agent:v2.3 .

# 或使用docker-compose
docker-compose up -d
```

### 4.2 docker-compose.yml

```yaml
version: '3.8'

services:
  javis-agent:
    image: javis-db-agent:v2.3
    ports:
      - "8000:8000"
      - "18080:18080"
    environment:
      - JAVIS_PG_HOST=postgres
      - JAVIS_PG_PORT=5432
      - JAVIS_PG_USER=postgres
      - JAVIS_PG_DATABASE=javis_db
      - OLLAMA_HOST=http://ollama:11434
    depends_on:
      - postgres
      - ollama

  postgres:
    image: postgres:15
    environment:
      - POSTGRES_DB=javis_db
    volumes:
      - postgres_data:/var/lib/postgresql/data

  ollama:
    image: ollama/ollama:latest
    volumes:
      - ollama_data:/root/.ollama

volumes:
  postgres_data:
  ollama_data:
```

## 五、systemd服务部署（Linux）

### 5.1 创建服务文件

```bash
sudo cat > /etc/systemd/system/javis-agent.service << EOF
[Unit]
Description=Javis-DB-Agent Service
After=network.target postgresql.service

[Service]
Type=simple
User=your_user
WorkingDirectory=/path/to/Javis-DB-Agent
ExecStart=/path/to/Javis-DB-Agent/.venv/bin/python -m uvicorn src.main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF
```

### 5.2 启动服务

```bash
# 重载systemd配置
sudo systemctl daemon-reload

# 启用开机自启
sudo systemctl enable javis-agent

# 启动服务
sudo systemctl start javis-agent

# 检查状态
sudo systemctl status javis-agent
```

### 8.1 配置文件位置

| 配置 | 路径 |
|------|------|
| 主配置 | `configs/config.yaml` |
| 工具定义 | `configs/tools.yaml` |

```yaml

  host: "0.0.0.0"
  port: 8000
  debug: false

  use_mock: true  # true=Mock模式, false=真实API

  type: "postgresql"
  host: "localhost"
  port: 5432
  name: "javis_db"

  host: "http://localhost:11434"
  model: "glm4"
```

## 九、卸载

### 9.1 停止服务

```bash
# systemd
sudo systemctl stop javis-agent
```

```bash
# 删除数据库
psql -U postgres -c "DROP DATABASE IF EXISTS javis_db;"

rm -rf .venv

rm -rf Javis-DB-Agent
```

## 十、故障排查

### 10.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 端口占用 | 8000端口被占用 | 修改config.yaml中的端口 |
| 数据库连接失败 | PostgreSQL未启动 | `brew services start postgresql` |
| Ollama连接失败 | 模型未下载 | `ollama pull glm4` |
| 导入错误 | 依赖未安装 | `pip install -r requirements.txt` |

### 10.2 日志位置

| 类型 | 位置 |
|------|------|
| systemd日志 | `journalctl -u javis-agent` |
| Docker日志 | `docker logs javis-agent` |
| 应用日志 | 控制台输出 |
