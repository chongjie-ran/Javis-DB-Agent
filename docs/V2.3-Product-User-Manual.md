## 一、产品概述

### 1.1 产品定位

**Javis-DB-Agent** 是面向数据库运维场景的智能体系统，赋能DBA和运维人员实现：
- 智能诊断与自动巡检
- 安全闭环处置
- 知识库辅助决策

### 1.2 核心价值

| 价值 | 说明 |
|------|------|
| 🔍 **智能诊断** | AI驱动的告警诊断，自动根因分析 |
| 🔒 **安全护栏** | SQL AST解析，危险操作审批流 |
| 📚 **知识积累** | SOP执行器，知识图谱辅助决策 |
| ✅ **受控闭环** | 从"建议输出"到"受控执行" |

## 二、核心功能

### 2.1 SQL安全护栏 (V2.1新增)

**功能**：基于SQL AST解析的安全校验，防止危险SQL执行。

**能力**：
- ✅ sqlglot AST解析（支持MySQL/PostgreSQL/Oracle/OceanBase）
- ✅ 危险操作检测（DROP/TRUNCATE/SHUTDOWN/GRANT）
- ✅ UNION注入防护
- ✅ DML边界检查（WHERE必填、LIMIT强制）
- ✅ 白名单模板匹配
- ✅ 只读命令识别（SELECT/EXPLAIN/ANALYZE）

**风险级别**：
| 级别 | 名称 | 说明 | 审批要求 |
|------|------|------|----------|
| L1 | 只读分析 | 查看数据、分析问题 | 无需 |
| L2 | 自动诊断 | 诊断、根因分析 | 无需 |
| L3 | 低风险执行 | 低风险操作 | 仅记录 |
| L4 | 中风险执行 | 中等风险操作 | 单签审批 |
| L5 | 高风险执行 | 高风险、破坏性操作 | 双人审批 |

### 2.2 SOP执行器 (V2.0新增)

**功能**：自动化执行标准操作规程，实现闭环处置。

**能力**：
- ✅ YAML格式SOP定义
- ✅ 22个Action→Tool映射
- ✅ 逐步骤执行与状态机
- ✅ PreCheck/PostCheck Hook
- ✅ 执行回流验证
- ✅ 审批联动（L4/L5）

**内置SOP**：
| SOP | 说明 |
|-----|------|
| slow_sql_diagnosis | 慢SQL诊断 |
| lock_wait_diagnosis | 锁等待诊断 |
| session_cleanup | 会话清理 |

### 2.3 ApprovalGate审批网关 (V2.1新增)

**功能**：高风险操作的审批流控制。

**能力**：
- ✅ L4单签审批（1人通过即可）
- ✅ L5双人审批（需2人通过）
- ✅ 异步审批请求
- ✅ 幂等性保障
- ✅ 超时处理
- ✅ REST API集成

**API端点**：
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/approvals/pending | 获取待审批列表 |
| GET | /api/v1/approvals/{id} | 获取审批详情 |
| POST | /api/v1/approvals/{id}/approve | 审批通过 |
| POST | /api/v1/approvals/{id}/reject | 审批拒绝 |

### 2.4 知识层增强

**功能**：知识图谱+案例库+RAG混合检索。

**能力**：
- ✅ 告警→症状→根因→动作推理链路
- ✅ 案例相似度检索
- ✅ Chroma向量存储
- ✅ 混合检索（向量+关键词）

### 2.5 感知层增强

**功能**：拓扑感知+配置感知。

**能力**：
- ✅ pg_stat_replication主从拓扑发现
- ✅ pg_settings参数采集
- ✅ 配置异常检测
- ✅ DirectPostgreSQL直连

## 三、使用指南

### 3.1 快速开始

```bash
# 1. 进入项目目录
cd ~/SWproject/Javis-DB-Agent

# 2. 激活虚拟环境
source .venv/bin/activate

# 3. 启动Mock API（开发测试用）
python3 -m uvicorn mock_javis_api.server:app --host 0.0.0.0 --port 18080 &

# 4. 启动FastAPI服务
python3 -m uvicorn src.main:app --host 0.0.0.0 --port 8000

# 5. 访问Dashboard
open http://localhost:8000/dashboard/
```

### 3.2 API调用示例

**告警诊断**：
```bash
curl -X POST http://localhost:8000/api/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "告警 ALT-001 是什么问题？"}'
```

**SQL分析**：
```bash
curl -X POST http://localhost:8000/api/v1/analyze/sql \
  -H "Content-Type: application/json" \
  -d '{"sql": "SELECT * FROM orders WHERE status = pending"}'
```

**健康巡检**：
```bash
curl -X POST http://localhost:8000/api/v1/inspect \
  -H "Content-Type: application/json" \
  -d '{"instance_id": "INS-001"}'
```

### 3.3 SOP执行

```python
from src.security.execution.sop_executor import SOPExecutor
from src.gateway.approval import ApprovalGate

# 创建执行器
executor = SOPExecutor(
    yaml_sop_dir="knowledge/sop_yaml/",
    approval_gate=ApprovalGate()
)

# 执行SOP
result = await executor.execute(
    sop_id="slow_sql_diagnosis",
    context={"instance_id": "INS-001"}
)
```

### 3.4 SQL护栏使用

```python
from src.security.sql_guard import SQLGuard, SQLGuardStatus

guard = SQLGuard()

# 校验SQL
result = guard.guard(
    sql="SELECT * FROM orders WHERE id = 1",
    context={"db_type": "postgresql"}
)

if result.status == SQLGuardStatus.DENIED:
    print(f"拒绝原因: {result.blocked_reason}")
elif result.status == SQLGuardStatus.NEED_APPROVAL:
    print("需要审批")
else:
    print("允许执行")
```

### 3.5 审批流程

```python
from src.gateway.approval import ApprovalGate

gate = ApprovalGate(timeout_seconds=300)

# 发起审批请求
request_id = await gate.request_approval(
    step_def={"action": "kill_session", "risk_level": "L4"},
    params={"pid": 12345}
)

# 审批
await gate.approve(request_id, approver="admin", comment="确认终止")
```

### 4.1 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| JAVIS_PG_HOST | localhost | PostgreSQL主机 |
| JAVIS_PG_PORT | 5432 | PostgreSQL端口 |
| JAVIS_PG_USER | postgres | PostgreSQL用户 |
| JAVIS_PG_PASSWORD | | PostgreSQL密码 |
| JAVIS_PG_DATABASE | postgres | 数据库名 |
| JAVIS_SOP_YAML_DIR | knowledge/sop_yaml/ | SOP YAML目录 |

### 4.2 API模式切换

```bash
# 查看当前模式
python scripts/switch_api_mode.py status

# 切换到Mock模式
python scripts/switch_api_mode.py mock

# 切换到真实API模式
python scripts/switch_api_mode.py real --api-key YOUR_KEY
```
