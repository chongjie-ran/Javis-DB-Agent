## 二、快速部署（Docker）

### 2.1 前置要求

- Docker 20.10+
- Docker Compose
- 8GB+ RAM

### 2.2 部署步骤

```bash
# 1. 克隆项目
git clone https://github.com/chongjie-ran/Javis-DB-Agent.git
cd Javis-DB-Agent

# 2. 配置
cp .env.example .env
# 编辑 .env 配置ollama地址

# 3. 启动（基础）
docker-compose up -d

# 4. 验证
curl http://localhost:8000/health
```

### 2.3 生产部署（MySQL）

```bash
# 启动完整服务（含MySQL）
docker-compose -f docker-compose.yml -f docker-compose.mysql.yml up -d

# 查看日志
docker-compose logs -f
```

## 三、Ollama部署

### 3.1 安装Ollama

```bash
# Linux/Mac
curl -fsSL https://ollama.com/install.sh | sh

# 验证安装
ollama --version
```

### 3.2 下载模型

```bash
# 推荐：qwen3.5:35b（高质量推理）
ollama pull qwen3.5:35b

# 备用：glm4（快速验证）
ollama pull glm4

# 备用：llama3.2（轻量级）
ollama pull llama3.2

# 查看已安装模型
ollama list
```

### 3.3 验证Ollama

```bash
curl http://localhost:11434/api/version

# 测试模型
ollama run qwen3.5:35b "你好"
```

## 四、源码部署

### 4.1 环境准备

```bash
# Python 3.10+
python3 --version

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 4.2 配置

```bash
# 复制配置
cp configs/config.yaml.example configs/config.yaml

# 编辑配置
vim configs/config.yaml
```

关键配置项：
```yaml
ollama:
  base_url: http://localhost:11434
  model: qwen3.5:35b
  timeout: 120

api:
  host: 0.0.0.0
  port: 8000
```

### 4.3 启动服务

```bash
# 启动API服务
python -m uvicorn src.main:app --host 0.0.0.0 --port 8000

# 或使用gunicorn
gunicorn -w 4 -k uvicorn.workers.UvicornWorker src.main:app --bind 0.0.0.0:8000
```

## 六、系统服务（Systemd）

### 6.1 创建服务文件

```bash
sudo vim /etc/systemd/system/javis-agent.service
```

内容：
```ini
[Unit]
Description=Javis-DB-Agent Service
After=network.target

[Service]
Type=simple
User=your_user
WorkingDirectory=/path/to/Javis-DB-Agent
ExecStart=/path/to/venv/bin/python -m uvicorn src.main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

### 6.2 启用服务

```bash
sudo systemctl daemon-reload
sudo systemctl enable javis-agent
sudo systemctl start javis-agent
sudo systemctl status javis-agent
```

## 七、负载均衡

### 7.1 Nginx配置

```nginx
upstream javis_backend {
    server 127.0.0.1:8000;
    keepalive 32;
}

server {
    listen 80;
    server_name javis.example.com;

    location / {
        proxy_pass http://javis_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 120s;
    }
}
```

### 7.2 HTTPS配置

```nginx
server {
    listen 443 ssl;
    server_name javis.example.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://javis_backend;
        # ... 其他配置
    }
}
```

## 八、备份与恢复

### 8.1 数据备份

```bash
# SQLite备份
cp data/javis_db_agent.db /backup/javis_$(date +%Y%m%d).db

# PostgreSQL备份
pg_dump -U javis_user -d javis_agent > /backup/javis_$(date +%Y%m%d).sql
```

### 8.2 恢复数据

```bash
# SQLite恢复
cp /backup/javis_20260403.db data/javis_db_agent.db

# PostgreSQL恢复
psql -U javis_user -d javis_agent < /backup/javis_20260403.sql
```

## 十一、常见问题

### 11.1 端口占用

```bash
# 查看端口占用
lsof -i:8000

# 杀掉进程
kill -9 <PID>
```

### 11.2 内存不足

增加Docker内存限制或添加Swap：
```bash
# 添加4GB Swap
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### 11.3 模型下载慢

使用代理或镜像：
```bash
export HTTP_PROXY=http://proxy:8080
export HTTPS_PROXY=http://proxy:8080
ollama pull qwen3.5:35b
```
