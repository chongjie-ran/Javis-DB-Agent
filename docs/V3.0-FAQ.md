## 一、安装部署问题

### Q1: Docker构建失败怎么办？

**A:** 常见原因及解决方案：

1. **内存不足**
   ```bash
   # macOS: Docker Desktop → Settings → Resources → 分配8GB+
   # Linux: 增加Swap
   sudo fallocate -l 4G /swapfile
   sudo chmod 600 /swapfile
   sudo mkswap /swapfile
   sudo swapon /swapfile
   ```

2. **网络问题**
   ```bash
   # 使用代理
   export HTTP_PROXY=http://proxy:8080
   export HTTPS_PROXY=http://proxy:8080
   docker-compose build --no-cache
   ```

### Q2: Ollama启动失败？

**A:** 
```bash
# 检查Ollama状态
ollama --version

# 启动Ollama服务
ollama serve

# 验证
curl http://localhost:11434/api/version
```

### Q3: 模型下载失败？

**A:**
```bash
# 设置代理
export HTTP_PROXY=http://proxy:8080
export HTTPS_PROXY=http://proxy:8080

# 重新下载
ollama pull qwen3.5:35b

# 查看进度
ollama list
```

## 二、配置问题

### Q4: 如何切换LLM模型？

**A:** 编辑 `configs/config.yaml`：
```yaml
ollama:
  model: qwen3.5:35b  # 推荐
  # model: glm4  # 备用
  # model: llama3.2  # 轻量
```

### Q5: 如何配置MySQL数据库？

**A:**
```yaml
database:
  type: postgresql
  host: localhost
  port: 5432
  name: javis_agent
  user: your_user
  password: your_password
```

### Q6: API超时怎么调整？

**A:**
```yaml
ollama:
  timeout: 120  # 增加超时时间（秒）
```

## 三、运行问题

### Q7: 服务启动后无法访问？

**A:**
```bash
# 1. 检查端口
lsof -i:8000

# 2. 检查日志
docker-compose logs javis-agent

# 3. 检查防火墙
sudo ufw allow 8000

# 4. 检查Ollama连接
curl http://localhost:11434/api/version
```

### Q8: LLM调用报超时？

**A:**
1. 检查Ollama服务是否运行：`curl http://localhost:11434/api/version`
2. 检查模型是否下载：`ollama list`
3. 增加超时配置
4. 考虑使用更小的模型

### Q9: 会话丢失怎么办？

**A:**
- 默认使用SQLite持久化，检查 `data/javis_db_agent.db` 是否存在
- 重启后会话自动恢复
- 检查磁盘空间是否充足

## 四、安全问题

### Q10: 如何配置Webhook签名验证？

**A:**
```python
import hmac, hashlib

def verify_webhook_signature(payload: bytes, signature: str, secret: str) -> bool:
    expected = hmac.new(secret.encode(), payload, hashlib.sha256).hexdigest()
    return hmac.compare_digest(f"sha256={expected}", signature)
```

### Q11: 如何设置IP白名单？

**A:**
```yaml
webhook:
  ip_whitelist:
    - "10.0.0.0/8"
    - "192.168.1.0/24"
```

## 五、性能问题

### Q12: 响应慢怎么优化？

**A:**
1. 使用更快的模型：`glm4` > `qwen3.5:35b`
2. 减少上下文长度
3. 启用缓存
4. 增加并发限制

### Q13: 内存占用高？

**A:**
```bash
# macOS: Docker Desktop限制内存
# Linux: 
docker update --memory=8g javis-agent

# 或减少Ollama模型大小
ollama run glm4  # 使用更小的模型
```

## 六、Hook开发问题

### Q14: 如何自定义Hook？

**A:**
```python
from src.hooks import Hook, HookEvent

class MyHook(Hook):
    async def trigger(self, event: HookEvent, context: dict) -> None:
        if event == HookEvent.BEFORE_LLM:
            # 自定义逻辑
            pass
```

### Q15: Hook执行失败会影响主流程吗？

**A:** 不会。V3.0的CompositeHook支持错误隔离，一个Hook失败不会影响其他Hook或主流程。

## 七、测试问题

### Q16: 如何运行测试？

**A:**
```bash
# 全部测试
python3 -m pytest tests/ -v

# 特定测试
python3 -m pytest tests/round30/ -v

# DFX测试
python3 -m pytest tests/round_dfx/ -v
```

### Q17: ChromaDB测试失败？

**A:** ChromaDB是可选依赖，未安装时会自动跳过相关测试。如需完整测试：
```bash
docker-compose up -d chroma
```
