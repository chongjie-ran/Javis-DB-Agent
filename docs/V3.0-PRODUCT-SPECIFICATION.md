## 一、产品概述

### 1.1 产品定位

**Javis-DB-Agent** 是面向数据库运维场景的本地AI智能体系统，赋能DBA和运维人员实现：
- 智能诊断
- 自动巡检
- 安全闭环处置

### 1.2 核心价值

| 价值点 | 描述 |
|--------|------|
| 🤖 智能化 | 基于大模型的智能诊断，减少人工干预 |
| 🔒 安全化 | 多层安全护栏，确保操作安全 |
| ⏱️ 高效化 | 自动巡检，问题快速响应 |
| 📚 知识化 | 经验积累，知识复用 |

## 三、核心功能模块

### 3.1 Hook生命周期系统 (Phase 0)

Hook是V3.0的核心扩展机制，提供9个生命周期注入点：

| Hook点 | 触发时机 | 用途 |
|--------|----------|------|
| `before_iteration` | 每次迭代开始前 | 前置检查、日志记录 |
| `before_execute_tools` | 工具执行前 | 参数验证、安全检查 |
| `after_execute_tools` | 工具执行后 | 结果处理、错误恢复 |
| `before_llm` | LLM调用前 | Prompt增强、上下文注入 |
| `on_stream` | LLM流式输出中 | 实时日志、中断处理 |
| `after_llm` | LLM调用后 | 响应处理、日志记录 |
| `after_iteration` | 每次迭代结束后 | 结果汇总、状态更新 |
| `on_error` | 发生错误时 | 错误处理、告警通知 |
| `on_complete` | 任务完成时 | 最终处理、资源清理 |

**CompositeHook**：支持多个Hook组合，错误隔离（一个Hook失败不影响其他）。

### 3.2 AgentRunner重构 (Phase 1)

重构Agent执行为规格化流程：

| 组件 | 说明 |
|------|------|
| `AgentRunSpec` | 执行规格定义（超时、token限制、允许工具等） |
| `AgentRunner` | 统一执行引擎 |
| `InstructionSelfContainValidator` | 指令自包含验证，确保Agent理解完整任务 |

### 3.3 双层记忆系统 (Phase 2)

| 记忆层 | 文件 | 说明 |
|--------|------|------|
| 短期记忆 | `HISTORY.md` | 事件日志，按时间顺序记录 |
| 长期记忆 | `MEMORY.md` | 结构化事实，持久化知识 |
| Token监控 | `TokenMonitor` | 实时监控token使用，防止溢出 |

### 3.4 对抗性验证系统 (Phase 3)

主动发现Agent推理中的问题：

| 组件 | 说明 |
|------|------|
| `ValidatorRegistry` | 验证器注册表 |
| `BreakingValidator` | 打破声明，验证Agent是否遵守约束 |
| `AdversarialValidationHook` | 对抗性验证Hook，自动触发 |

### 3.5 自我合理化防护 (Phase 4)

防止Agent自我合理化跳过检查：

| 功能 | 说明 |
|------|------|
| 19个信号词检测 | "看起来正确"、"应该没问题"等 |
| 9种防护动作 | 警告、重试、强制验证等 |
| 开发者特有信号 | 代码模式、调试输出检测 |

### 3.6 TaskScheduler (Phase 5)

智能任务调度与死锁预防：

| 功能 | 说明 |
|------|------|
| 任务分类 | READ_ONLY / WRITE_SAME_FILE / WRITE_DIFF_FILE / VERIFY |
| FileLockManager | 文件锁管理，防止并发冲突 |
| 死锁预防 | 按路径排序，避免循环等待 |

### 3.7 Subagent双模式 (Phase 6)

| 模式 | 超时 | 权限 | Token限制 | 用途 |
|------|------|------|-----------|------|
| ExploreSpec | 5分钟 | 只读 | 30k | 探索分析 |
| ExecuteSpec | 60分钟 | 可写 | 100k | 执行操作 |

## 四、安全机制

### 4.1 SQL安全护栏

| 级别 | 风险 | 处理 |
|------|------|------|
| L1-L3 | 低-中 | 自动执行 |
| L4 | 高 | 需要审批 |
| L5 | 极高 | 强制阻断 |

### 4.2 Webhook审批

- HMAC-SHA256签名验证
- IP白名单
- Replay攻击防护

### 4.3 会话安全

- Token TTL过期机制
- 参数漂移检测
- 并发访问控制
