## 1. 设计原则

### 1.1 核心原则
- **安全第一**：LLM不直接生成SQL/Shell，所有操作经受控工具层
- **工具化能力**：数据库运维能力全部封装为Tool API
- **多Agent协同**：统一编排Agent调度专业Agent
- **上下文驱动**：以zCloud平台真实运维上下文为主

### 1.2 架构主张
```
不是：LLM → SQL → DB
而是：LLM → Tool Intent → 参数约束 → 安全校验 → Tool Executor → DB
```

## 3. 核心组件

### 3.1 Agent Gateway
参考OpenClaw Gateway架构，负责：
- **Session管理**：多会话隔离、上下文管理
- **Tool注册**：工具的注册、发现、调用
- **Agent编排**：协调多Agent协作
- **策略引擎**：安全策略、权限控制
- **审计日志**：完整操作链路记录

### 3.2 统一编排Agent
```python
class OrchestratorAgent:
    """统一编排Agent"""
    def process(goal: str, context: Context) -> Plan:
        # 1. 解析用户目标
        # 2. 选择专业Agent
        # 3. 组织上下文
        # 4. 调度执行
        # 5. 汇总结果
```

### 3.3 专业Agent矩阵

| Agent | 核心能力 | Tool依赖 |
|-------|----------|----------|
| 诊断Agent | 根因分析、置信度输出 | QueryTools, 知识库 |
| 风险评估Agent | 风险分级、处置建议 | AnalysisTools |
| SQL分析Agent | SQL画像、锁分析 | QueryTools, AnalysisTools |
| 巡检Agent | 健康评分、风险项 | QueryTools, ActionTools |
| 报告生成Agent | 结构化报告 | 所有Tools |
| 处置决策Agent | SOP匹配、审批路由 | ActionTools, 知识库 |

### 3.4 工具层架构

## 4. 数据流设计

### 4.1 告警诊断流程
```
1. 告警事件 → Gateway
2. 编排Agent接收 → 解析告警类型
3. 选择诊断Agent + 风险Agent
4. 诊断Agent调用:
   - QueryTools获取上下文
   - 知识库匹配告警规则
5. 风险Agent评估影响范围
6. 汇总诊断结果 + 风险建议
7. 用户确认 → 处置执行或结束
```

### 4.2 SQL分析流程
```
1. 用户输入/告警触发 → Gateway
2. 编排Agent → 选择SQL分析Agent
3. SQL分析Agent调用:
   - QueryTools获取SQL详情
   - AnalysisTools分析执行计划
   - 知识库匹配优化规则
4. 输出: 慢SQL原因 + 优化建议 + 风险评估
```

## 5. 技术架构要点

### 5.1 多Agent协同模型
- 采用**编排-执行**分离模式
- 统一编排Agent负责任务分解和结果聚合
- 专业Agent专注于特定领域

### 5.2 上下文管理
- 短期上下文：会话级，Agent推理用
- 长期上下文：知识库，结构化运维知识
- 实时上下文：zCloud平台数据

### 5.3 LLM交互模式
- System Prompt：定义Agent角色和约束
- Tool Definitions：结构化工具描述
- JSON Mode：结构化输出
- 安全校验：参数和结果双校验

## 6. 与OpenClaw架构的借鉴关系

| OpenClaw | zCloud Agent | 说明 |
|----------|--------------|------|
| Gateway | Agent Gateway | 会话管理、工具注册、策略引擎 |
| Session | Context | 多会话隔离、上下文管理 |
| Tool Registry | Tool Executor | 工具的注册、发现、调用 |
| Agent (main/design/qa) | 多Agent矩阵 | 不同专业Agent |
| Skills | 知识库+SOP | 专家知识封装 |

## 7. 演进策略

### 本期（Copilot阶段）
- 单Agent为主，编排简单
- 工具集为查询和知识检索
- 不开放执行类工具

### 下期（受控闭环）
- 多Agent协同
- 开放低风险执行工具
- 引入审批流

### 远期（自治运维）
- 复杂多Agent协同
- 策略驱动的自动化
- 完整审计追溯
