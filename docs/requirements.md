# Javis-DB-Agent 需求文档

> 版本：v1.0 | 日期：2026-03-28 | 状态：初稿

---

## 1. 项目背景

### 1.1 目标
构建面向**数据库运维场景**的本地智能体系统，赋能DBA和运维人员实现智能诊断、自动巡检与安全闭环处置。

### 1.2 核心约束
- **LLM基础**：Qwen3.5（Ollama，本地运行，适配64GB显存）
- **部署形态**：本地化部署，数据不出网
- **参考架构**：zCloud数据库运维智能体架构
- **参考实现**：OpenClaw架构

---

## 2. 核心功能需求

### 2.1 感知与上下文层

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 监控数据接入 | 实时获取实例状态、CPU/内存/IO、会话数、锁等待 | P0 |
| 告警事件接入 | 接收zCloud告警事件，触发Agent分析 | P0 |
| 拓扑信息获取 | 获取实例-数据库-主机-存储关系 | P1 |
| SQL画像接入 | Top SQL、慢SQL、执行计划、等待事件 | P1 |
| 运维历史获取 | 工单记录、变更记录、历史故障 | P2 |

### 2.2 知识与语义层

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 运维知识库 | 基础知识、版本特性、参数说明、故障模式 | P0 |
| 告警知识库 | 告警定义、触发逻辑、排查路径 | P0 |
| SOP/Runbook库 | 日常巡检、故障响应、性能排查标准流程 | P1 |
| 案例库 | 历史故障、根因、处置过程、结果 | P1 |
| 知识图谱 | 实例-告警-SQL-参数关系建模 | P2 |

### 2.3 智能决策层（多Agent协同）

| Agent | 职责 | 优先级 |
|-------|------|--------|
| 统一编排Agent | 解析用户目标、选择专业Agent、组织上下文、汇总结果 | P0 |
| 诊断Agent | 根因分析、输出诊断路径、置信度 | P0 |
| 风险评估Agent | 风险级别、是否可自动处置 | P0 |
| SQL分析Agent | SQL画像、慢SQL、锁阻塞、优化建议 | P1 |
| 巡检Agent | 健康评分、风险项、治理建议 | P1 |
| 报告生成Agent | RCA报告、巡检报告摘要 | P1 |
| 处置决策Agent | SOP匹配、是否转人工审批 | P2 |

### 2.4 工具执行层

| 类型 | 工具 | 优先级 |
|------|------|--------|
| 查询类 | 实例状态、参数、锁等待、复制状态、会话、慢SQL | P0 |
| 分析类 | 诊断报告、SQL归因、告警关联 | P1 |
| 执行类 | 触发巡检、低风险SOP、刷新采样 | P2 |
| 联动类 | 发通知、创建工单、审计记录 | P1 |

### 2.5 安全治理层

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 权限分层 | 查看→分析→建议→执行→高危执行 | P0 |
| 动作分级 | L1只读分析 → L4需审批 → L5禁止 | P0 |
| SQL护栏 | 只允许SQL子集、模板校验、limit超时 | P0 |
| 审批确认 | 高风险必须人工/双人审批 | P1 |
| 审计留痕 | 完整操作链路追踪 | P0 |

## 3. 优先落地场景（6个）

| 场景 | 输入 | 输出 | 阶段 |
|------|------|------|------|
| 告警解释与根因诊断 | 告警ID/事件 | 告警含义+排查路径+置信度 | Copilot |
| 慢SQL/锁阻塞分析 | SQL指纹/会话ID | 原因分析+优化建议+风险评估 | Copilot |
| 巡检与健康治理 | 巡检任务 | 健康评分+风险项+治理优先级 | Copilot |
| 参数与配置风险 | 实例ID | 配置偏差+参数冲突+风险项 | Copilot |
| 复制/HA状态诊断 | 实例ID | 主从延迟+复制状态+切换风险 | Copilot |
| 报告生成 | 任务ID | RCA报告/巡检报告/领导摘要 | Copilot |

## 4. 演进路线

### 阶段一：Copilot辅助（本期）
- 能看、能问、能解释、能建议
- 只读为主、不自动执行、用户决策

### 阶段二：受控闭环（下期）
- 能决策、能调用工具、能做低风险动作
- Tool-based、风险分级、人机协同

### 阶段三：自治运维（远期）
- 场景自治、策略自治、可审计自治

---

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| 响应时间 | 单次诊断 < 30秒 |
| 本地LLM | Qwen3.5 (Ollama) |
| 显存需求 | 适配64GB显存 |
| 数据安全 | 本地部署，数据不出网 |
| 可用性 | 7x24 告警响应 |
| 审计要求 | 全操作链路留痕 |
