# HA切换处理 SOP

> 版本：v1.0 | 更新：2026-03-28 | 适用：主备切换/故障切换应急处理

---

## 1. HA切换类型

| 类型 | 触发原因 | 处理优先级 |
|------|----------|------------|
| 计划切换 | 维护/升级 | 低（已规划） |
| 故障切换 | 主库不可达 | 高（紧急） |
| 人工切换 | 管理员手动 | 中 |
| 自动切换 | 监控系统触发 | 高（需确认） |

---

## 2. 切换后立即确认

### 2.1 确认业务状态
```
□ 确认应用是否可以连接新主库
□ 确认读写分离是否正常
□ 确认是否有报错的应用
□ 确认是否有数据丢失
```

### 2.2 确认数据库状态
```sql
-- 查看当前实例角色
SELECT database_role FROM v$database;

-- 查看HA状态
SELECT * FROM v$dg_configuration;

-- 查看最近切换时间
SELECT value FROM v$parameter WHERE name = 'cluster_database_instances';
```

---

## 3. 故障切换排查

### 3.1 确认原主库状态
```sql
-- 尝试连接原主库
SELECT status FROM v$instance;

-- 查看原主库告警日志
-- （需要登录原主库主机查看alert日志）
```

### 3.2 确认切换原因
```
排查方向：
1. 原主库实例崩溃（ORA-600/ORA-7445）
2. 网络中断
3. 存储故障
4. 主机硬件故障
5. 资源配置问题（内存/磁盘）
```

### 3.3 数据一致性检查
```sql
-- 检查是否有数据丢失（PostgreSQL）
SELECT pg_last_wal_receive_lsn(), pg_last_wal_replay_lsn();

-- 检查复制完整性
SELECT * FROM pg_stat_replication;

-- 估算数据差异
SELECT now() - pg_last_xact_replay_timestamp() AS replication_delay;
```

## 4. 切换后处理

### 4.1 短期处理（0-4小时）
```
□ 确认应用连接正常
□ 确认数据无丢失或评估丢失范围
□ 提升监控频率
□ 记录切换事件
□ 通知相关方
```

### 4.2 原主库恢复（4-24小时）
```
□ 排查原主库故障根因
□ 修复原主库
□ 评估是否需要回切
□ 回切风险评估
□ 如不回切，将原主库作为新备库加入
```

### 4.3 回切决策
```
回切条件：
□ 原主库完全修复
□ 确认无数据丢失
□ 业务低峰期
□ 获得相关方批准

不回切条件：
□ 原主库硬件损坏
□ 回切风险大于收益
□ 业务已在新主库稳定运行
```

## 5. 切换报告

### 5.1 必填信息
```json
{
  "切换时间": "2026-03-28 10:30:00",
  "切换类型": "故障切换",
  "原主库": "INS-001 (192.168.1.10)",
  "新主库": "INS-002 (192.168.1.11)",
  "触发原因": "原主库实例崩溃 ORA-00600",
  "数据丢失": "无",
  "业务中断时间": "30秒",
  "处理人": "DBA-张三",
  "影响范围": "订单系统写入短暂失败"
}
```

### 5.2 RTO/RPO评估
```
RTO（恢复时间目标）：业务中断时长
RPO（恢复点目标）：数据丢失量

HA切换目标：
- RTO < 5分钟
- RPO < 1分钟
```

## 6. 预防措施

| 措施 | 说明 |
|------|------|
| 定期切换演练 | 每季度至少一次 |
| HA监控 | 实时监控主备状态 |
| 容量规划 | 避免因容量问题触发切换 |
| 变更评审 | 重大变更评估HA影响 |
| 根因分析 | 每次切换后必须有根因报告 |
