# 巡检标准流程 SOP

> 版本：v1.0 | 更新：2026-03-28 | 适用：日常巡检

---

## 1. 巡检准备

### 1.1 巡检类型
| 类型 | 检查范围 | 时长 |
|------|----------|------|
| quick | 核心指标 | ~1分钟 |
| full | 全量检查 | ~5分钟 |
| security | 安全检查 | ~3分钟 |

### 1.2 巡检触发时机
- 每日定时（建议9:00前完成）
- 变更前后
- 故障恢复后
- 重要业务上线前

---

## 2. 巡检执行步骤

### 2.1 第一步：实例健康检查
```
检查项：
□ 实例状态是否UP
□ 实例启动时间
□ 实例角色（主/从/备）
□ HA状态
□ 当前连接数 / 最大连接数
□ 告警数量（严重/警告/提示）
```

### 2.2 第二步：资源使用检查
```
检查项：
□ CPU使用率（<80%正常，>80%关注，>90%告警）
□ 内存使用率（<85%正常，>85%关注，>95%告警）
□ 磁盘使用率（<80%正常，>80%关注，>90%告警）
□ 表空间使用率
□ 临时表空间使用率
□ 归档空间使用率
```

### 2.3 第三步：性能指标检查
```
检查项：
□ 当前活跃会话数
□ 阻塞会话数
□ 锁等待数量
□ 最近15分钟慢SQL数量
□ Top SQL（按执行时间）
□ Top SQL（按逻辑读）
□ Top SQL（按物理读）
□ Buffer Cache命中率
□ 库缓存命中率
```

### 2.4 第四步：复制与HA检查
```
检查项：
□ 主从复制状态（正常/中断/延迟）
□ 复制延迟秒数（<30s正常，>30s关注）
□ 最近HA切换时间
□ 备库健康状态
□ 备份任务状态
```

### 2.5 第五步：安全与变更检查
```
检查项：
□ 最近参数变更记录
□ 最近DDL变更
□ 失败登录尝试
□ 权限变更审计
```

## 3. 健康评分模型

### 3.1 评分维度（100分制）
| 维度 | 权重 | 评分规则 |
|------|------|----------|
| 可用性 | 25% | 实例UP=25，实例DOWN=0 |
| 资源使用 | 25% | 100-使用率（综合CPU/内存/磁盘） |
| 性能 | 25% | 无慢SQL=25，有则扣分 |
| 复制 | 15% | 复制正常=15，延迟>30s=5，中断=0 |
| 安全 | 10% | 无异常=10，有告警扣分 |

### 3.2 健康等级
| 评分 | 等级 | 说明 |
|------|------|------|
| 90-100 | 优秀 | 正常运行 |
| 75-89 | 良好 | 轻微问题，关注即可 |
| 60-74 | 一般 | 存在风险，建议处理 |
| 40-59 | 较差 | 存在严重风险，优先处理 |
| 0-39 | 危险 | 需要立即处理 |

## 5. 巡检后处理

### 5.1 风险处理优先级
1. **P0（立即处理）**：健康等级危险，实例不可用
2. **P1（24小时内）**：健康等级较差，资源告警
3. **P2（本周内）**：健康等级一般，优化建议
4. **P3（下周内）**：健康等级良好，小问题

### 5.2 通知规则
- P0/P1风险：立即发送告警通知
- P2风险：日报汇总
- P3风险：周报汇总
