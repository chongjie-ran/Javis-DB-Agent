# 数据库高负载排查标准流程 (SOP)

## 1. 告警确认

### 1.1 收到告警
- 告警类型: CPU_HIGH / MEMORY_HIGH / INSTANCE_SLOW
- 记录告警时间、实例ID
- 检查资源使用率具体数值

### 1.2 初步判断
```
CPU > 80%: CPU高负载
CPU > 95%: 紧急处理
内存 > 85%: 内存压力
响应时间 > 3秒: 严重性能问题
```

## 2. 信息收集

### 2.1 查看当前会话
```sql
-- MySQL: 查看当前连接和执行的SQL
SHOW FULL PROCESSLIST;

-- PostgreSQL: 查看当前会话
SELECT * FROM pg_stat_activity WHERE state != 'idle';

-- Oracle: 查看当前会话
SELECT sid, serial#, username, status, sql_id, event, seconds_in_wait
FROM v$session WHERE status = 'ACTIVE';
```

### 2.2 查看慢查询
```sql
-- MySQL: 查看慢查询
SELECT * FROM mysql.slow_log ORDER BY start_time DESC LIMIT 10;

-- 查看未完成查询
SELECT * FROM information_schema.innodb_trx;
```

### 2.3 查看系统资源
```bash
# CPU使用详情
top -bn1 | head -30

# 内存使用
free -h

# 磁盘IO
iostat -x 1 3

# 网络连接状态
netstat -an | grep 3306 | wc -l
```

### 2.4 查看锁等待
```sql
-- MySQL: 查看锁等待
SELECT * FROM information_schema.innodb_lock_waits;

-- 查看InnoDB状态
SHOW ENGINE INNODB STATUS;
```

## 3. 问题定位

### 3.1 CPU高负载
**常见原因:**
- 大量复杂查询
- 缺少索引导致全表扫描
- 排序和分组操作
- 并发请求过多

**处理步骤:**
1. 识别高CPU的SQL
2. 查看执行计划
3. 添加索引或优化SQL
4. 限制并发

### 3.2 内存高负载
**常见原因:**
- Buffer Pool配置不当
- 大查询占用内存
- 内存泄漏
- 连接数过多

**处理步骤:**
1. 检查Buffer Pool使用率
2. 查看大查询
3. 调整内存参数
4. 限制单查询内存

### 3.3 响应缓慢
**常见原因:**
- 锁等待
- 慢查询
- 网络延迟
- 资源耗尽

**处理步骤:**
1. 检查锁等待
2. Kill长时间运行的查询
3. 检查网络
4. 重启服务(紧急情况)

## 4. 紧急处置

### 4.1 识别问题SQL
```sql
-- MySQL: 查找执行时间最长的查询
SELECT id, user, host, db, command, time, state, info
FROM information_schema.processlist
WHERE command != 'Daemon'
ORDER BY time DESC LIMIT 5;
```

### 4.2 Kill问题会话
```sql
-- MySQL: Kill会话
KILL [session_id];

-- 批量Kill慢查询
SELECT CONCAT('KILL ', id, ';')
FROM information_schema.processlist
WHERE command != 'Daemon'
  AND time > 60;
```

### 4.3 限制新连接
```sql
-- 设置最大连接数
SET GLOBAL max_connections = 50;

-- 等待紧急恢复
```

## 5. 恢复验证

### 5.1 确认负载下降
```bash
# 监控资源使用率
watch -n 2 'free -h; top -bn1 | head -10'
```

### 5.2 确认业务恢复
- 检查应用响应时间
- 确认无报错
- 监控业务指标

## 6. 后续处理

### 6.1 问题分析
- 分析根本原因
- 评估影响范围
- 制定优化方案

### 6.2 预防措施
- 添加必要索引
- 优化SQL
- 调整参数
- 设置更严格的监控

### 6.3 记录归档
- 记录故障时间线
- 保存相关日志
- 更新知识库
- 复盘改进
