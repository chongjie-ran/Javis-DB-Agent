# V2.2 迭代规划

> 创建时间：2026-03-31
> 承接版本：V2.1

---

## 一、背景与目标

V2.1 完成了核心安全架构建设（ApprovalGate、SOP YAML化、直连PG），V2.2 聚焦**稳定性收尾**与**性能提升**。

核心目标：
1. 修复 API 路径双重前缀问题
2. 清理旧 ApprovalGate 遗留代码
3. 完善文档
4. 性能优化

---

## 二、迭代内容

### 2.1 API 双重前缀修复（最高优先级）

**问题描述**：
approval 相关路径出现 `/api/v1/api/v1/...` 双重前缀

**涉及文件**：
- `src/main.py` - 路由注册
- `src/api/approval_routes.py` - 路径定义
- `src/gateway/approval.py` - 旧 ApprovalGate

**解决思路**：
- 统一路径前缀规范（API_ROUTER_PREFIX 或直接用 ""）
- 确保 approval_router 注册时不再额外加前缀

---

### 2.2 旧 ApprovalGate 迁移/清理

**问题描述**：
新 ApprovalGate (`src/gateway/approval.py`) 已实现，旧实现可能仍有残留

**涉及文件**：
- `src/gateway/` - 清理旧实现
- `src/api/approval_routes.py` - 确认只用新版
- `src/main.py` - 确认旧版已移除

**清理项**：
- [ ] 删除旧的 `src/gateway/approval_old.py` 或备份
- [ ] 确认所有审批走新 ApprovalGate
- [ ] 确认 `src/main.py` 中无旧版引用

### 2.4 性能优化

**潜在优化点**：
- DirectPostgresConnector 连接池管理
- SOP YAML 加载缓存（避免每次重启重复解析）
- API 响应压缩
- ChromaDB 查询优化

**评估指标**：
- API P99 响应时间 < 500ms
- SOP 加载时间 < 100ms
