## 失败分类

### 🔴 FIX_NEEDED（测试代码BUG，共13个）

#### F1: `test_int_01_002` — 危险SQL未被编排器拦截
- **问题**: 编排器返回 `success=True, error=''`，测试期望 `success=False` 或 error 包含 "blocked"
- **根因**: `OrchestratorAgent._process_direct()` 不调用 `sql_guard`，直接将任务委托给 `SQLAnalyzerAgent`，SQL分析器返回风险评估（risk_level=L5）但仍标记 `success=True`
- **涉及文件**: `tests/v2.0/test_integration.py:67`
- **建议修复**: 测试应验证 orchestrator 最终拒绝执行，而不是期望 `success=False`；或者在编排器层面集成 sql_guard 护栏

#### F2: `test_int_01_003` — AgentResponse 不支持 `.get()` 方法
- **问题**: `result.get('approval_required', False)` 抛出 `AttributeError: 'AgentResponse' object has no attribute 'get'`
- **根因**: `AgentResponse` 是 dataclass，不是 dict
- **涉及文件**: `tests/v2.0/test_integration.py:86`
- **建议修复**: `result.metadata.get('approval_required', False)` 或直接断言 `result.success` 状态

#### F3: `test_int_02_001` — knowledge_graph fixture 返回同步 MagicMock
- **问题**: `await knowledge_graph.add_node(...)` 抛出 `TypeError: 'MagicMock' object can't be awaited`
- **根因**: `knowledge_graph` fixture（conftest.py:380）返回 `MagicMock`（同步），但测试中 `add_node` 是 async 方法
- **涉及文件**: `tests/v2.0/conftest.py:380` + `tests/v2.0/test_integration.py:102`
- **建议修复**: 将 fixture 中的 `MagicMock` 替换为 `AsyncMock`

#### F4: `test_int_02_002` — case_library fixture 返回同步 MagicMock
- **问题**: `await case_library.add_case(...)` 抛出 `TypeError: 'str' object can't be awaited`
- **根因**: `case_library` fixture（conftest.py:390）返回 `MagicMock`（同步）
- **涉及文件**: `tests/v2.0/conftest.py:390` + `tests/v2.0/test_integration.py:131`
- **建议修复**: 将 `add_case` mock 改为 `AsyncMock`

#### F5: `test_int_03_001` — DiagnosticAgent.diagnose() 方法不存在
- **问题**: `AttributeError: 'DiagnosticAgent' object has no attribute 'diagnose'`
- **根因**: `DiagnosticAgent` 有 `diagnose_alert()` / `diagnose_instance()` / `diagnose_alert_chain()`，但没有 `diagnose()` 方法
- **涉及文件**: `tests/v2.0/test_integration.py:173`
- **建议修复**: 改用正确的 `diagnose_alert()` 或 `diagnose_instance()` 方法

#### F6: `test_int_03_002` — config_tools fixture 返回 dict（同步）但被 await
- **问题**: `await config_tools.get_instance_config(...)` 抛出 `TypeError: 'dict' object can't be awaited`
- **根因**: `config_tools` fixture（conftest.py:418）返回 `MagicMock`，其 `get_instance_config` 不是 async mock
- **涉及文件**: `tests/v2.0/conftest.py:418` + `tests/v2.0/test_integration.py:191`
- **建议修复**: 将 `get_instance_config` 改为 `AsyncMock`

#### F7: `test_int_04_001` — PolicyResult 被 await 但不是协程

- **根因**: `mock_policy.check.return_value = PolicyResult(...)` 返回的是普通对象而非协程
- **涉及文件**: `tests/v2.0/conftest.py:488` + `tests/v2.0/test_integration.py:229`
- **建议修复**: `mock_policy.check` 本身是同步的，不应该被 await；测试中应移除 `await`
